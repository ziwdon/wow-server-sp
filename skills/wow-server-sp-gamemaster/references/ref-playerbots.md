# Playerbot Reference (mod-playerbots)

## Overview

This server uses the **mod-playerbots fork** of AzerothCore (`mod-playerbots/azerothcore-wotlk`, branch `Playerbot`), NOT the canonical `azerothcore/azerothcore-wotlk`.

Two types of bots:
- **Altbots** — characters you create manually and add via `.playerbots bot add`; they do NOT auto-gear or roam
- **Rndbots** (Random Bots) — auto-generated by the server (`RNDBOT*` accounts); auto-gear, auto-talent, roam the world

The rndbot pool size is configured at install time and varies per installation. Check the actual configured values:
```bash
grep -E "AC_AI_PLAYERBOT_(MIN|MAX)_RANDOM_BOTS" /opt/stacks/azerothcore/docker-compose.override.yml
```
Pool = `RNDBOT*` accounts × up to 10 characters each. The installer default is 250 accounts (≈2500 characters), but the user may have chosen a different number.

**Key default behavior:** `AC_AI_PLAYERBOT_DISABLED_WITHOUT_REAL_PLAYER=1` — rndbots are disabled when no real player is logged in. When you log in, the engine ramps bots toward `AC_AI_PLAYERBOT_MIN_RANDOM_BOTS`.

## Bot Setup Commands

### Altbot Setup
```
.playerbots bot add [name1,name2,name3]     # Login altbots (characters on your account)
.playerbots bot addaccount [accountname]    # Login all characters on an account
.playerbots bot remove name1,name2,name3   # Logout altbots
.playerbots bot add *                       # Login all altbots in your party/raid
.playerbots bot remove *                    # Logout all altbots in party/raid
```

### AddClass Bots (borrow from rndbot pool)
```
.playerbots bot addclass [className]    # Summon an rndbot of the desired class
# Class names: warrior, paladin, hunter, rogue, priest, shaman, mage, warlock, druid, dk
```

### Bot Maintenance (whisper to bot, or /p for party, /r for raid)
```
maintenance     # Learn spells, supplement consumables, enchant gear, repair
autogear        # Auto-equip best gear (quality based on AutoGearQualityLimit)
talents         # Show current spec
talents spec list              # List available specs for this class
talents spec [spec name]       # Force spec change
talents apply <link>           # Apply specific talents link
glyphs                         # Show equipped glyphs
glyph equip [ID1 ID2 ID3 ID4 ID5 ID6]   # Apply specific glyphs
reset botAI                    # Reset bot settings
reset                          # Reset current bot actions
```

## Party / Raid General Commands

```
summon          # Summon bot to you
release         # Release spirit when dead
revive          # Revive near a spirit healer
leave           # Leave party
attack          # Attack selected target
follow          # Run toward player
flee            # Run toward player ignoring everything
stay            # Stay in place
grind           # Attack anything visible
give leader     # Pass party/raid leader
disperse set 8  # Force bots to spread out, maintaining 8 yards between each other
disperse disable  # Reset disperse distance to default
```

> `disperse set x` is useful in raids to prevent AoE/chain abilities from hitting clustered bots (e.g., `@ranged disperse set 6` before Akil'zon or Anub'Rekhan).

### Group Targeting
Target specific groups/roles for commands:
```
@group1 follow          # Command group 1 to follow
@tank attack            # Command all tanks to attack
@dps attack             # Command all DPS to attack
@heal follow            # Command all healers to follow
@ranged follow          # Ranged only
@meleedps attack        # Melee DPS only
@rangeddps attack       # Ranged DPS only
@[className] follow     # Target by class (e.g., @druid, @mage)
@group1,4 follow        # Multiple groups
@group2-5,8 follow      # Range + specific groups
```

## Strategy System

```
co +strategy,-strategy,~strategy    # Modify combat strategies
nc +strategy,-strategy              # Modify non-combat strategies
co ?                                # Show current combat strategies
nc ?                                # Show current non-combat strategies
co !                                # Reset combat strategies
nc !                                # Reset non-combat strategies
```

### Key Combat Strategies
| Strategy | Description |
|----------|-------------|
| `tank` | Use threat-generating abilities |
| `tank assist` | Tank pulls mobs off others |
| `dps` | Use DPS abilities |
| `cc` | Use CC abilities (needs RTI cc target, default moon icon) |
| `assist` | Target one mob at a time |
| `aoe` | Target many mobs at once |
| `boost` | Use major cooldowns (useful to toggle on bosses: `co -boost`, then `co +boost`) |
| `threat` | DPS actively avoid grabbing threat |
| `grind` | Attack any visible target |
| `heal` | Focus on party healing |
| `focus` | Stop AoE; focus on single target |
| `avoid aoe` | Automatically avoid harmful AoE |
| `save mana` | Healers prioritize high-efficiency spells when low mana |
| `healer dps` | Healers cast damage spells when mana allows |
| `tank face` | Tank doesn't face ranged players (counters breath/cleave) |
| `behind` | Move to target's back |
| `pull` | Tank bot pulls with ranged skill |
| `pull back` | Tank pulls and backs to starting position |
| `wait for attack` | Bots wait N seconds before engaging (set via `wait for attack time 5`) |

### Key Non-Combat Strategies
| Strategy | Description |
|----------|-------------|
| `food` | Initiate or stop eating/drinking |
| `pvp` | Toggle PvP mode |
| `loot` | Enable looting (or use with `grind`) |

### Class-Specific Strategies

**Druid** (auto-set by talent selection; can override):
`bear` — tank spec, `cat` — melee DPS, `caster` — moonkin/resto

**Hunter**:
- `trap weave` — drop Explosive Trap in rotation
- Non-combat: `bdps` (Hawk), `bspeed` (Pack/Cheetah), `bmana` (Viper), `rnature` (Wild)

**Mage**:
- `frost` / `fire` — spec-specific (auto from talents)
- `firestarter` — Fire mages go melee to trigger instant Flamestrike

**Paladin** (extremely useful in raids):
- Blessings: `bdps` (Might), `bmana` (Wisdom), `bstats` (Kings), `bhealth` (Kings/Sanctuary)
- Auras: `rfire`, `rfrost`, `rshadow`, `baoe` (Retribution), `barmor` (Devotion), `bcast` (Concentration), `bspeed` (Crusader)

**Priest** (non-combat):
- `rshadow` — cast Shadow Protection on party

**Shaman** (combat):
- `co +tremor` / `co +fire` / `co +[totem name]` — add a specific totem to Call of the Elements

**Warlock** — Pet selection (non-combat):
`imp`, `voidwalker`, `succubus`, `felhunter`, `felguard` — force specific pet
- Soulstone: `ss master`, `ss self`, `ss tank`, `ss healer`
- Combat: `meta melee` — Demonology only; go melee during Metamorphosis

## Targeting / RTI / RTSC

### RTI (Raid Target Icons)
```
rti <icon>          # Set bot's priority target icon (skull, cross, circle, star, square, triangle, diamond, moon)
attack rti target   # Bots attack their RTI target
rti cc <icon>       # Set CC target icon (default: moon)
```

### RTSC (Right-click targeting system — point-and-click positioning)
```
rtsc                        # Toggle on + give you the "aedm" spell
rtsc cancel                 # Toggle off + remove spell
rtsc save [#]               # Save a location (1-9) when you cast aedm on it
rtsc unsave [#]             # Clear saved location
rtsc go [#]                 # Command bots to move to saved location
@tank rtsc go 5             # Send all tanks to location 5
rtsc go save                # Return bots to their saved positions
[name/group] rtsc toggle    # Toggle point-and-click for specific bots
```

### Focus Heal
```
focus heal +[PlayerName]    # Add player to focus heal list
focus heal -[PlayerName]    # Remove player
focus heal ?                # Show current focus heal list
focus heal clear            # Clear all
```

## Performance Configuration

### Key env vars (in docker-compose.override.yml)

| Env Var | Recommended | Description |
|---------|-------------|-------------|
| `AC_AI_PLAYERBOT_ENABLED` | `1` | Enable playerbots |
| `AC_AI_PLAYERBOT_MIN_RANDOM_BOTS` | installer-set | Min rndbots to maintain (check override.yml) |
| `AC_AI_PLAYERBOT_MAX_RANDOM_BOTS` | installer-set | Max rndbots (check override.yml) |
| `AC_AI_PLAYERBOT_DISABLED_WITHOUT_REAL_PLAYER` | `1` | Disable rndbots when no real player online |
| `AC_AI_PLAYERBOT_BOT_ACTIVE_ALONE` | `10` | % chance idle bots become active (10=best for performance) |
| `AC_AI_PLAYERBOT_BOT_ACTIVE_ALONE_SMART_SCALE` | `1` | Auto-scale activity based on latency |
| `AC_MAP_UPDATE_THREADS` | `4` | Map update threads |

### Performance Profiles

**Profile 1 — Best performance (recommended for this server):**
```
AiPlayerbot.BotActiveAlone = 10
AiPlayerbot.botActiveAloneSmartScale = 1
AiPlayerbot.botActiveAloneSmartScaleWhenMinLevel = 1
AiPlayerbot.botActiveAloneSmartScaleWhenMaxLevel = 80
```
Bots stay idle when not in the same zone. Best choice for performance with any bot count when you don't need bots active everywhere.

**Profile 2 — 100% activity with auto-scale:**
```
AiPlayerbot.BotActiveAlone = 100
AiPlayerbot.botActiveAloneSmartScale = 1
```
Good if you want bots everywhere at all times (more CPU/RAM).

### Verify performance
```bash
# In-game:
.server info    # Check server diff time. Should be < 70-80ms baseline
                # Percentiles should max out around 100-150ms
```

### Bot Activity Rules (in order)
Regardless of `BotActiveAlone`, bots are **always active** when:
1. In a battleground, instance, or raid
2. In combat (defending themselves)
3. In a zone with real players
4. In a guild with real players
5. Has a real player as master
6. Grouped with a real player as leader
7. In battleground queue
8. Looking for group
9. Within 300 yards of a real player
10. On a real player's friend list
11. More than 10 other bots are nearby (force spread out)

## Expected Bot States

### "Random Bots Stats: 0 online" — **This is normal** when no real player is logged in.
`AC_AI_PLAYERBOT_DISABLED_WITHOUT_REAL_PLAYER=1` keeps bots offline when no real player is connected. Characters are **not deleted** — they're just `online=0` in the DB. When you log in, the engine ramps bots toward `AC_AI_PLAYERBOT_MIN_RANDOM_BOTS`.

### Verify bot pool integrity
```bash
docker exec -i ac-database mysql -uroot -p"$DOCKER_DB_ROOT_PASSWORD" \
    -e "SELECT COUNT(*) FROM acore_characters.characters c \
        JOIN acore_auth.account a ON a.id=c.account \
        WHERE a.username LIKE 'RNDBOT%'"
# Expected: configured_accounts × chars_per_account (installer default = 250 accounts × 10 chars = ~2500)
# Your actual configured bot count:
grep -E "AC_AI_PLAYERBOT_(MIN|MAX)_RANDOM_BOTS" /opt/stacks/azerothcore/docker-compose.override.yml
```

## Troubleshooting Playerbots

**Bots don't respond to commands:**
- Check `AiPlayerbot.Enabled = 1` is set
- Module must be compiled (check `.playerbots bot list`)
- EnUS DBC files required server-side

**Bots stuck at level 1:**
- Let server run a full cycle (check update intervals in playerbots.conf)
- Check `AiPlayerbot.RandomBotAutologin = 1`
- Check server logs for SQL or pathfinding errors

**Server becomes laggy with bots:**
1. Increase `MapUpdate.Threads` (to 4-6)
2. Reduce bot count
3. Set `BotActiveAlone = 10`
4. Monitor with `.server info` in-game

**Bots crash in dungeons/raids:**
- Check for module conflicts
- Update to latest playerbots
- Report with full logs and stack traces

**A:<action> - FAILED messages in Playerbots.log:**
- This is **normal and benign** — these are action-retry traces logged every tick
- High volume is expected; not errors

For more: see `docs/wikis/mod-playerbots-wiki/Troubleshooting.md`

## Macro Collection

Ready-to-use macros for common bot control scenarios.

### Controlled pull (requires Slashin or ElvUI addon for `/in` command)
```
/p @dps co +passive
/p @heal co +passive
/p @tank attack
/in 8 /p @dps co -passive
/in 8 /p @heal co -passive
```
Lets the tank build aggro for 8 seconds before DPS and healers engage.

### Suppress Bloodlust/Heroism
```
# Stop bots using BL/Hero:
/p @shaman ss +2825,32182

# Re-enable:
/p @shaman ss -2825,32182
```

### Flee together (pull back from danger)
```
/p reset
/p nc -stay,+follow,+passive
/p co +passive
/p do follow
```

### Follow and assist (resume normal combat)
```
/p nc -stay,+follow,-passive
/p co -passive
/p do follow
```

### Stay and hold position
```
/p nc -follow,+stay,-passive
/p co +passive
/p do stay
```

### Focus bots on a specific creature (e.g., Web Wrap)
```
/target Web Wrap
/stopmacro [noharm][dead]
/script SetRaidTarget("target", 8)
```
Sets the diamond icon on the target; use `rti diamond` to have bots prioritize it.
